# New blog post

Creates `blog/<slug>.html` from the template and lists it on `blog/index.html`. Both steps are required — a post that isn't in the index is unreachable.

## Ben writes the prose

Ben's standing rule for this blog, stated in his first post: **every word is written by his own hand, with nothing generated by AI.** Honor that.

Do not draft, ghostwrite, or "improve" post prose. Do not fill body paragraphs with placeholder or invented text. Set up the scaffolding and let him write into it.

What you do: create the file, wire up the structure, apply the formatting conventions, fix markup mistakes, and add the index row. If he pastes or dictates prose, place it verbatim — fixing only markup, not wording. If a post arrives with body text missing, leave the body paragraphs empty for him rather than inventing them.

## Steps

1. **Gather** the title, the date (default today), the hashtags, and the prose if he has it. Ask only for what's missing and actually needed.

2. **Pick the slug** — lowercase, hyphenated, derived from the title (`Day One` → `day-one.html`). Confirm it doesn't already exist in `blog/`.

3. **Copy `blog/_template.html`** to `blog/<slug>.html`. Start from the template rather than an existing post, so stale content can't leak in. Read it first — its comments carry the conventions and are the source of truth if this file disagrees.

4. **Fill the placeholders:**
   - `<title>` — keep the ` | Ben Keil` suffix.
   - `<meta name="description">` — one real sentence. Not a placeholder; it's what search results and link previews show.
   - `<h1>` and the `.post-header` `<h1>` title text.
   - `<time datetime="YYYY-MM-DD">` and its visible text — keep the two in sync.

5. **Apply the opening-line rule.** If the first paragraph is *exactly one sentence*, it stays as `<p class="post-lead">` (centered). If it is two or more sentences, delete that `<p>` entirely and let the paragraph start inside `.post-body`, left-aligned. Sentence count is the whole test — length doesn't matter, and a multi-sentence paragraph is never centered.

6. **Format the body** — one `<p>` per paragraph inside `.post-body`. No inline styles. No `<br>` for spacing. `<strong>` for the first mention of a key topic. Link text reads as words, not raw URLs (`creditcard.cafe`, not `https://creditcard.cafe`).

7. **Hashtags stay separate** — one per `<li>` in the `<ul class="post-tags">` below the body, lowercase, `#` included. Never a trailing line of prose in the last paragraph.

8. **Add the index row.** In `blog/index.html`, inside `<section class="blog-list">`, newest first:

   ```html
   <article class="blog-preview">
     <h3>
       <a href="<slug>.html"><title></a>
     </h3>

     <p class="post-meta"><Month D, YYYY></p>

     <p class="blog-excerpt">
       <opening of the post>
     </p>

     <a class="read-more" href="<slug>.html">Read more &rarr;</a>
   </article>
   ```

   The title link and the "Read more" link must point at the same file — that has been wrong before. `.blog-excerpt` is CSS-clamped to two lines, so don't hand-type a trailing "…".

9. **Hand it to Ben to look at.** He checks rendering himself; don't launch a browser or screenshot it.

## Don't

- Don't wrap post content in `class="section"` — it's a full-viewport centered panel (`min-height: 100vh`, `text-align: center`) and will center the prose. `.post` and `.blog-list` exist for this reason.
- Don't rename `blog/_template.html` or strip its leading underscore — Jekyll skips underscore-prefixed files, which is what keeps it off the live site.
